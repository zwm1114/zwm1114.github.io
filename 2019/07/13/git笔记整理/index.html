廖雪峰Git教程笔记整理

2019-07-13

几个常用命令：

查看哪些文件修改了：git status

查看修改的内容：git diff

查看提交日志：git log

简化日志信息：git log --pretty=oneline

一、版本管理

1.把当前版本回退到上一个版本

git reset --hard HEAD^

git //查看文件内容

2.回退到上一个版本之后又想回到这个版本

找到该版本号（无需写全，只需写前几位，多写几位不然找不到的…）

git reset --hard

3.当前窗口已关闭，还想回到某一个版本

git reflog //找到版本id

git reset --hard

二、工作区与版本库

理解他们很重要!

commit 后，用 git diff HEAD -- 查看工作区和版本库里最新版本的区别。

1.没使用add和commit

git checkout -- //丢弃文件的修改内容

2.使用了add ，未使用 commit

git reset HEAD //相当于撤销掉了add命令

3.使用了add和commit命令

版本回退

三、删除文件

add和commit 之后想删除文件

1.确实想删除

git rm

git commit -m “XXX”

2.手滑

git checkout -- //一键还原，就是用版本库里的版本替换工作区的版本

ps:从未添加进版本库里的被删除的文件是无法恢复的!

四、远程仓库

1.初始化本地仓库

git init

2.添加远程仓库

git remote add <SSH/Https>

3.把本地库的内容推送到远程库

如果远程库是空的：git push -u //第一次推送，本地分支与远程分支关联起来

非空：git push //提交

4.从远程库克隆

git clone <SSH/Https> //推荐使用SSH

五、新建分支

1.创建分支

git checkout -b

2.查看当前分支

git branch

3.切换回某个分支

git checkout

4.合并分支

git merge

ps:这种合并是快速合并，另外有 --no-ff方式合并分支

git merge --no-ff -m ‘XXX’

5.合并完分支可以删除分支

git branch -d //删除分支

六、解决冲突

1.当两个分支都有了各自的提交

需要手动修改不同的内容，然后add，commit

ps: git log --graph --pretty=oneline --abbrev -commit

七、分支管理策略

1.最佳实践：

1）master分支十分稳定，用来发布新版本

2）dev是用来工作的分支

3）团队成员有自己的分支，往dev上合并

2.bug分支

git stash //保护现场

git checkout //选择在哪个分支上修复bug

git checkout -b //创建分支

git checkout //切换回原来的分支

git merge --no-ff -m ‘XXX’ //采用--no-ff方式合并分支

git branch -d //合并完，删除分支

git checkout //切换到原来工作的分支上

git stash list //查看现场

//恢复现场

1）

git stash apply //恢复

git stash drop //删除stash内容

2）

git stash pop //恢复的同时把stash内容也删除了

ps:可以多次恢复，然后统一drop

//恢复指定内容

git stash //查看

git stash apply stash @{0} //恢复指定内容

//丢弃一个没有合并过的分支

git branch -D

八、多人协作

git remote //查看远程看的信息

git remote -v //显示更详细的信息

git push origin //推送分支

git checkout -b / //创建远程的dev分支到本地

？两人提交的代码有冲突怎么办

git pull 会失败

git branch --set-upstream-to = / //指定本地branch与远程branch连接

git pull //再pull

九、Rebase

先pull别人的然后再git rebase。

作用：好看！把本地未push的分叉提交历史整理成直线，使查看历史提交变化时更容易。

十、标签管理

1.创建标签

git checkout //先切换到该分支上

git tag //打标签

git tag //查看所有标签

ps:默认标签是打在最新提交的commit上

若想在之前的commit上打标签，先找到历史提交的commit id

git tag

git show //查看标签信息

git tag -a -m ‘XXX’ //创建带有说明的标签

ps:标签总是和某个commit挂钩的，如果这个commit出现在两个不同的分支，那么这两个分支都可以看到这个标签

2.操作标签

git tag -d //删除标签

git push //推送某个标签到远程

git push --tags //推送全部未推送到远程的本地标签

若标签已推送到远程

git tag -d //先本地删除

git push :refs/tags/ //从远程删除

其他：git remote rm //删除已有的github远程库

十一、自定义git

1.忽略特殊文件

没懂…

2.配置别名

git config --global alias.

例如：git config --global alias.lg “log --graph --pretty=oneline --abbrev-commit”

​ 输入git lg 即可查看分支合并情况

配置信息在隐藏目录 .git/config 中

ps:global参数是全局参数，也就是这些命令在这台电脑的所有git仓库下都有用

十二、个人遇到的错误及解决办法

refusing to merge unrelated histories* 本地仓库和远程不是同一个仓库，无法合并两个独立的仓库历史

解决办法：git pull --allow-unrelated-histories
