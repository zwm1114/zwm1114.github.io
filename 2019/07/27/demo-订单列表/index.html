1.项目搭建

npm start

3.目录说明

index.js文件时整个项目的入口文件，在该文件中调用了App组件

ReactDOM.render(<App />, document.getElementById('root'));

ReactDOM.render()是用于将模板转为 HTML 语言，并插入指定的 DOM 节点，public 文件中的index.html中定义了id为‘root’的div元素，这里将App组件插入进div中。

在App组件中调用其他的组件。

4.组件间的数据通信

这里创建了三个组件： Header OrderItem OrderList

Header是渲染整个页面头部部分，它没有和其他组件进行通信，进行下样式的调整即可。

OrderList是渲染出整个页面的订单列表，每个列表由OrderItem渲染出，OrderList是OrderItem的父组件，相对地，OrderItem是OrderList的子组件。

OrderList/index.js

OrderList组件在constructor中，获取父组件的props,并且初始化state。

在componentDidMount（）中，使用fetch获取mock数据，在public中新建文件夹mock,里面新建文件存放mock数据。获取成功的话，返回一个promise对象，再将里面的数据取出，用setState更新data。

在render函数中调用OrderItem组件，将data通过props传递给OrderItem组件。通过handleSubmit将数据提交给后端（因为没有后端接口，与后端交互的功能暂时没有实现），同时更新state中的data数据。

OrderItem/index.js

同理，先获取父组件的props,并初始化state.

editing判断是否出现评价编辑框。

初始化stars为父组件传过来的数值（已评价）或者为0（未评价）。

同理，comment亦是。

在render中把获取的数据渲染到页面上，注意到，这里通过ifcommented来判断是否可以评价，如果未评价过，点击评价按钮，通过handleEditArea更新editing的值。

再回到render中，editing初始化为false，所以不显示评价文本框，当更新了state之后，editing为true,出现评价文本框。用renderEditArea来显示文本框。

文本框中默认显示的是state中comment的内容，当输入内容时，调用handleCommentChange，更新comment。

评价星星那里用renderStars显示。主要代码：

通过数组的map方法，显示出五个星星，点击的时候，通过handleClickStars更新stars的值。

再回到renderEditArea中，点击提交按钮，调用handleCommentSubmit函数，更新editing的值为false,这时就关闭了评价按钮了，并把参数传递给onSubmit（还记得OrderList中调用组件OrderItem时写了个onSubmit属性吗？onSubmit的属性值是一个函数，这个函数获取参数，把参数提交给后端）。

若点击取消按钮，则调用handleCommentCancle函数，将state值恢复为初始值。

和后端交互主要是在OrderList组件中，可以通过fetch与后端进行交互。
