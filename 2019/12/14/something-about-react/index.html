something about react

“粉条，最近的react学得咋样啊？”

“我不想学了”

“为什么呢？”

“因为我觉得我用html css javaScript用着挺顺手的，干嘛非得学这个？”

“不能半途而废啊！”

“我不管，除非你能用充分的理由说服我。”

“好吧。去准备点瓜子吧。”

“好嘞，爷。”

“首先你知道什么是react吗？”

“额…我看网上的解释是一个用于构建用户界面的javaScript库。”

“知道这句话什么意思吗？”

“额，字都认识，连在一起就不懂是什么意思了。”

“哪里不懂？”

“构建用户界面我懂，就是写网页嘛，就是和我用html css写出的网页是一样的，还非得说得文邹邹的，净整些没用的。”

“是啊，所以建议你以后别看百度了，直接谷歌，那些英文解释反而更好理解。是哪里不懂呢？”

“javaScript库是什么？我家的车库装的是车，JavaScript库装的是哪个牌子的车？”

“……JavaScript牌的。”

“哈哈哈哈，这个牌子的车好开吗？”

“挺好开的，毕竟那么多人开它。你知道JavaScript是干嘛用的吗？”

“写文章的那支笔。”

“额…还记着呢。你打开淘宝主页，那些图片，还有输入框，以及整个页面的排版，这些基础的静态结构是用HTML和css写出的。”

“你要在输入框中输入你想搜索的东西，知道这是谁负责处理的吗？”

“JavaScript？”

“没错，JavaScript就是让网页活起来的那支点睛之笔。你家车库里的车多吗？”

“多呀，有自行车还有自行车还有自行车以及自行车…..”

“都是同一种类型的吗？”

“不是哦，有山地车，公路车，折叠车，还有旅行车还有…”

“是挺多的…”

“JavaScript库里面也是有很多车吗？”

“是的。猜猜为什么在库里装那么多车？”

“嗯，是为了避免重复的工作。比如说，我想周末骑车出去玩，我就骑山地车，速度快，但是如果我平时上学我就骑公路车，不同的场景下我骑不同的车，我告诉你哦，我的第一辆公路车我想把他速度加快点，就自己改装，然后他就被我爸卖去了废品站了。”

“你记住下次能花钱买的，就别自己动手，挺费钱的。”

“……”

“在编写网页的时候，你想要实现一个功能，这个功能之前已经有人实现过了，而且就放在javaScript库中，你就别自己重复造轮子了，时间还是挺值钱的，当然，如果你对别人是如何实现这个功能很感兴趣的话，你可以研究研究源码。”

“谢谢，我不感兴趣。”

“JavaScript库里面就有一辆，不是，是一个名为react js的库。”

“是他很好用还是其他的库不好用？”

“一半一半吧，在react js出生之前，前端开发者写页面是纯手工的，或者是用像jQuery这样不注重ui的框架，说到jQuery，他也算是react js的爸爸了。这样开发页面开发周期长而且也更容易出现错误和bug。”

“哦……原来如此，可是我写页面也没有感觉原生JavaScript不好用啊？”

“你都开发了什么页面？”

“把我的全家福放在网页上展示了。”

“……不怪你，以后开发大型项目的时候，你用原生JavaScript写我就敬你是条汉子。”

“那为啥react js 这么霸道呢？”

“因为他有两大杀器：JSX和virtual DOM！”

“virtual DOM我知道，你昨天和我说过，那JSX又是什么呢？”

“昨天说过了可以用JavaScript来更改DOM从而动态的展示网页是不是？”

“嗯，是。”

“JSX的英文名是 short for JavaScript eXtension,现在能猜到是什么意思了吗？”

“就是JavaScript的精简版呗？”

“额，其实不全是。看看这个代码，就是用JSX写的。”

“额，有点奇怪哎，像JavaScript又像HTML。”

“是的。”

……

“没了？”

“没了。”

“今天就说这些了？”

“嗯，回家喝鱼汤了。”
